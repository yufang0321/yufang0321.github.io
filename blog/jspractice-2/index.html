淺拷貝(shallow copy)與深拷貝(deep copy)

By 芳芳 04 January 2024

在工作中踩過的雷，紀錄一下

淺拷貝shallow copy

方法一
const objA = {
      a: 1,
      b: 2
}

const objB = objA
objB.a = 3

console.log('objA.a:', objA.a) // 3
方法二：使用ES6的Rest Operator複製
const objA = {
  a: 1,
  b: {
    c: 2
  }
}

const objB = { ...objA }

objB.a = 3
objB.b.c = 4

console.log(objA) //{a:1, b:{c: 4}}
console.log(objB) //{a:3, b:{c: 4}}

可以注意到，上面的例子，objB.a時因為是第一層，objA不會被影響到 但objB.b.c時第二層之後，就會改道objA的值

方法三：Object.assign(target, source)

深拷貝deep copy

方法一：JSON.parse(JSON.stringify())
const objA = {
      a: 1,
      b: { c: 3 }
}

const objB = JSON.parse(JSON.stringify(objA))

console.log(objA === objB) // false
console.log(objA.b === objB.b) // false

另外參考：https://www.programfarmer.com/articles/2021/javascript-shallow-copy-deep-copy 有說到JSON.stringfy/parse處理後，有些值會產生變化，例如：

  • undefined：會連同key一起消失
  • NaN：會被轉成 null。
  • Infinity：會被轉成 null。
  • regExp：會被轉乘 空 {}。
  • Date：型別會由 Data 轉成 string。
