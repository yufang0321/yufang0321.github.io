函式陳述式與函式表達式

By 芳芳 06 January 2024

javascript書籍閱讀筆記，把不熟悉的觀念用自己的方式記錄與整理一遍！

先看看這一段程式碼，想想會印出甚麼~

var fn1 = function() {
    console.log("我是函式表達式");
}

function fn1() {
    console.log("我是函式陳述式");
}

fn1();

答案是：“我是函式表達式”

第一次看這段程式碼時，百思不得其解，主要有幾個重要觀念：

  1. 函式陳述式與函式表達式：
  • 函式陳述式(Function Statement)：直接宣告一個函式，並且賦與名稱。
  • 函式表達式(Function Expression)：宣告一個變數，使用運算子將函式附與到變數上。
  1. 函式陳述式優先：瀏覽器在在執行程式時，會經過兩階段，第一是創造階段、第二是執行階段
  • 創造階段時，會先準備一個記憶體空間
  • 執行階段時，則是真的把值賦與至變數上

而函式陳述式在創造階段就會提升，優於一般變數，故在創造階段後即可呼叫函式 因此上述程式碼真正的流程是

//創造階段：
function fn1() {
    console.log("我是函式陳述式");
}
var fn1;

//執行階段
fn1 = function() {
    console.log("我是函式表達式");
}

fn1();

在創造階段時，因為函式陳述式優先，故會先在創造階段時就定義好fn1函式，但此時尚未賦與值，在進入執行階段時，才會再把函式表達式內容賦與進去，故得到"我是函式表達式"的結果。
